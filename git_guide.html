Branching

Make a new branch:

git checkout -b "new-branch-name"

Make a new branch based on a specific upstream:

git checkout -b "new-branch-name" "upstream_name"

Switch branches:

git checkout "branch-name"

Change upstream tracking branch:

git checkout "branch-name" --set-upstream-to "new-upstream-name"

Revert to a Working Version

Revert to a specific commit hash:

git log -1

Change the number for however many commits back you want to see the hash for, then:

git reset --hard "commit-hash"

Hard reset to the tip of the branch:

git reset --hard

Handling Merge Conflicts

Resolve the conflicts as you wish, i.e. with ide or merge tool then run:

git merge --continue

This will ensure you don't accidentally alter the git blame for other people's commits.

Squash all Local Commits into One

You can squash lots of commits into a single one with a new message.

git merge --squash

Stash and Pop Commits

Stash will save your commits and let you pull without pushing first, pop then puts them back as local commits ready for pushing.

git stash
git stash pop
