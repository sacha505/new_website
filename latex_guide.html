Compile a Document

Single .tex file:

pdflatex document.tex

Alternatively, you can use something online like Overleaf.

Include Packages

All packages should to be included before the \begindocument command

\documentclass[12pt, letterpaper]{report}
\usepackage
\title{Your Title}
\begin{document}
\end{document}

Create a Contents Page

This is fairly simple:

\tableofcontents

Add a Bibliography

The most organized way to do this is to use BibTex. You can manage your bibliography manually but it's a pain.

You'll need to make a .bib file which will contain the each reference in your paper. You need to make sure these are accurate.
Each individual reference will look something like this:

@online{website_reference_1, author = "Jane Doe",
title = "My Book",
url = "https://www.wikipedia.org/",
addendum = "(accessed: 01.01.2010)",
keywords = "latex, books, guides"}

The first part of the reference denotes what type of document the reference is, in this case it is a website, hence @online but it could be something else like @book or @article
The document type determines the next fields you'll need for a proper reference. This site is en excellent guide to the fields you need to use: https://www.bibtex.com/e/entry-types/

To use the reference, call it next to the text block you want to cite using the name you assigned in bibtex.

\cite{website_reference_1}

Adding and Positioning Images

To add an image, it needs to be in a known directory to the interpreter. One common mistake is not to have included the right file path.

The easiest way to position images is by using float, represented here with the [H]:

\begin{figure}[H]
\centering
\includegraphics[width=0.5\linewidth]{pathtoyour/file/image.png}
\caption{your caption here}
\end{figure}

Float tells LaTeX to put your image in the same place it appears in the code. You can also see here there is a width modifier which you can use to resize your image.

Adding Multiple Images to a Page

When you begin a figure enviroment as shown above, you can then add subfigures to that environment to position multiple images on the same page.

\begin{figure}
\centering
\begin{subfigure}[b]{0.3\textwidth}
\includegraphics[scale=0.2]{image}
\caption{Text A}
\label{fig:desc}
\end{subfigure} %
\begin{subfigure}[b]{0.3\textwidth}
\includegraphics[scale=0.2]{image}
\caption{Text B}
\label{fig:desc}
\end{subfigure} %
\\
\begin{subfigure}[b]{0.3\textwidth}
\includegraphics[scale=0.2]{image}
\caption{Text C}
\label{fig:desc}
\end{subfigure} %
\begin{subfigure}[b]{0.3\textwidth}
\includegraphics[scale=0.2]{image}
\caption{Text D}
\label{fig:desc}
\end{subfigure} %
\end{figure}

That \\ tells LaTeX to put the next set of things you're doing on a new line.

Figures Within Figures

Figures can be subfigures of one larger figure, each of which having its own subheading. This can be nice for comparing two images with some limited description.

\begin{figure}[h] \begin{subfigure}{0.5\textwidth}
\includegraphics[width=0.5\linewidth, height=20cm]{image1.png}
\caption{caption1}
\label{fig:subim1}
\end{subfigure}
\begin{subfigure}{0.5\textwidth}
\includegraphics[width=0.5\linewidth, height=15cm]{image2.png}
\caption{caption2}
\label{fig:subim2}
\end{subfigure}
\caption{Caption 2}
\label{fig:image2}
\end{figure}

Adding Code Snippets

The \lstset environment lets us write example code

\usepackage{listings}
\begin{lstlisting}[language=C]
%your code here
\end{lstlisting}

Lists and Sublists

Plenty of ways to make lists in LaTeX, and you can nest lists inside of lists. It depends on your document type as to how many you can nest, but to my knowledge you can always nest three times in any type.

\begin {itemize}
\begin {itemize}
\begin {itemize}
\end {itemize}
\end {itemize}
\end {itemize}
